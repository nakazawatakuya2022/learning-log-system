# データ辞書（logs）

Status: Draft  
Last Updated: 2026-02-08

## logs テーブル

- **id**
    - 主キー
    - ログを一意に識別するID
- **occurred_at**
    - 学習日
    - 学習を行った日付
- **language**
    - プログラム言語
    - 例：PHP / Java / C++ など
    - 一覧・検索の絞り込みに使用
- **level**
    - ログの分類
    - NOTE / QUESTION / IMPORTANT
    - 意味：メモ／質問／重要事項
- **title**
    - タイトル
    - 学習ログの要点を表す
    - **必須**（一覧性・検索性のため）
- **message**
    - 学習過程
    - 内容：問題 / エラー / 試したこと
    - **必須**
- **solution**
    - 解決方法の内容（本文）
    - 内容：原因 / 解決 / 学び
    - 任意（未解決時は NULL 可）
- **is_resolved**
    - 解決状態フラグ
    - 0：未解決
    - 1：解決済み
    - **解決したかどうかの状態を表す**
- **is_knowledge**
    - ナレッジ扱いフラグ
    - 0：通常ログ
    - 1：ナレッジ
    - 再利用価値があるかどうかの判断用
- **created_at**
    - 作成日時
    - ログ初回登録日時
- **updated_at**
    - 更新日時
    - 編集／解決状態変更／ナレッジ切替時に更新

is_resolved ＝ 状態（解決したか）
solution     ＝ 内容（どう解決したか）
is_knowledge ＝ 価値（残すべきか）
