# 要件定義（Requirements）

Status: Draft  
Last Updated: 2026-02-08

---

## 1. プロジェクト概要

### 1.1 目的
- 学習ログを記録・整理し、一覧／検索／詳細表示で振り返れるようにする
- 学習内容を言語化し、解決方法をナレッジとして蓄積する
- 将来的な共有・Q&A拡張を見据えた基盤を作る

### 1.2 対象ユーザー
- 赤羽校 生徒
- MVP段階では単一ユーザー利用を前提とする

---

## 2. システム構成（MVP）

### 2.1 画面構成
本システムは **2画面構成** とする。

| 画面 | URL | 概要 |
|---|---|---|
| トップ | /logs | 新規登録＋一覧表示＋検索 |
| 詳細 | /logs/{id} | 詳細表示・編集・解決追記 |

※ `/logs/new` や `/edit` 専用画面は作成しない  
※ MVPとして最小構成・実装容易性を優先する

---

## 3. 機能要件

### 3.1 学習ログ登録機能
- 学習ログを新規登録できる
- 未解決の状態でも登録可能
- 登録時に以下の項目を入力できる

**必須項目**
- 学習日（occurred_at）
- 言語／技術（language）
- 分類（level：NOTE / QUESTION / IMPORTANT）
- タイトル（title）
- 学習過程（message）

**初期値**
- is_resolved：0（未解決）
- is_knowledge：0
- solution：NULL

---

### 3.2 学習ログ一覧表示・検索機能
- 登録済み学習ログを一覧表示できる
- 表示順は以下を採用する


- 最近「更新・整理」されたログを優先表示する

**一覧表示項目**
- 学習日
- 言語／技術
- 分類
- 状態（解決／未解決）
- タイトル

**検索条件**
- 学習日（From / To）
- 言語／技術
- 分類
- 未解決のみ
- 解決済みのみ
- ナレッジのみ

※ 未解決・解決済みが両方未選択の場合は全件表示  
※ solution の有無は検索条件・判定には使用しない

---

### 3.3 学習ログ詳細表示機能
- 学習ログの詳細情報を表示できる

**表示内容**
- タイトル
- 言語／技術
- 学習日
- 分類
- 更新日時
- 学習過程（message）
- 解決内容（solution）

※ solution 未記入時は「未記入」または非表示とする

---

### 3.4 学習ログ編集機能
- 詳細画面内で編集可能とする
- 「Edit」ボタン1つで **全項目を編集可能**
- 各項目ごとの編集ボタンは設けない

**編集対象**
- title
- message
- solution
- is_resolved
- is_knowledge

**編集操作**
- Cancel：編集開始時の状態に完全復元
- Save：まとめて保存

---

## 4. 状態・ナレッジの考え方（重要）

### 4.1 状態・内容・価値の分離
本システムでは以下の3要素を明確に分離する。

| 項目 | 意味 |
|---|---|
| is_resolved | 解決したかどうか（状態） |
| solution | 解決方法・学びの本文（内容） |
| is_knowledge | 再利用価値があるか（価値） |

- 解決判定に solution の有無は使用しない
- 未解決ログであってもナレッジ化は可能
- 解決済みであってもナレッジ化しない選択は可能

---

## 5. 保存方針
- solution 未記入時は NULL とする
- 解決状態の判定は常に is_resolved を使用する
